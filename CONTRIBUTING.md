# Contributing to Luanet

Thank you for your interest in contributing to Luanet! We welcome contributions from everyone.

## Getting Started

1. **Fork the repository** on GitHub.
2. **Clone your fork** locally:
   ```bash
   git clone https://github.com/yourusername/luanet.git
   cd luanet
   ```
3. **Create a new branch** for your feature or bugfix:
   ```bash
   git checkout -b feature/my-new-feature
   ```

## Development Workflow

1. **Make your changes**. Ensure your code is clean and readable.
2. **Add tests** for any new functionality or bug fixes.
3. **Run tests** to ensure no regressions:
   ```bash
   lua tests/test_tensor.lua
   ```
   (Add more tests as appropriate)

## Code Style

- Follow standard Lua coding conventions.
- Use 4 spaces for indentation.
- Keep lines reasonable length.
- Document your functions and modules.

## Submitting a Pull Request

1. **Push your branch** to your fork:
   ```bash
   git push origin feature/my-new-feature
   ```
2. **Open a Pull Request** on the main repository.
3. Provide a clear description of your changes and why they are necessary.
4. Link to any relevant issues.

## Reporting Issues

If you find a bug or have a feature request, please open an issue on GitHub. Include as much detail as possible, such as:
- Steps to reproduce the issue.
- Expected behavior.
- Actual behavior.
- Lua version and environment details.

Thank you for contributing!
